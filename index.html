❓ What is HTML?

HTML stands for HyperText Markup Language. It is the standard language for creating webpages.

🔗 Concept of Hyperlink and Markup

A hyperlink is a link that connects one page to another. Markup refers to the use of tags to structure and present content.

Example:

<a href="https://www.wikipedia.org">Go to Wikipedia</a>

Output:

Go to Wikipedia

🚫 HTML is Not a Programming Language

HTML is a markup language, meaning it describes the structure of data rather than performing logic like a programming language does.

🌍 World's First Website

The first website was created by Tim Berners-Lee in 1991. It is still live: http://info.cern.ch .

🔡 UTF-8 & ASCII

What is Character Encoding?

Character encoding is a system that translates characters, like letters and symbols, into a series of numbers that computers can understand. Think of it as a dictionary where each character is assigned a unique code.

ASCII

ASCII (American Standard Code for Information Interchange) is a classic character encoding that uses 7 bits to represent 128 characters. This includes the English alphabet (A-Z, a-z), numbers (0-9), and basic symbols like punctuation.

For example, in ASCII, the letter 'A' is represented by the number 65.

Because it only uses 7 bits, ASCII has a major limitation: it cannot represent characters from other languages, such as Chinese, Cyrillic, or Arabic, or even modern symbols and emojis.

UTF-8

UTF-8 (Unicode Transformation Format) is the most widely used character encoding on the web today. It can represent every character in the Unicode standard, which includes characters from almost every writing system in the world.

A key feature of UTF-8 is that it is a variable-width encoding. This means it can use between 1 and 4 bytes to represent a character. For English characters, it uses a single byte, just like ASCII. For more complex characters and emojis, it uses more bytes.

Because of this, UTF-8 is fully backward compatible with ASCII. Any document written in pure ASCII is also a valid UTF-8 document. This is why it has become the universal standard for web development.

🧠 Semantic Elements

<header>, <footer>, <main>, <section>, <article> are semantic elements introduced in HTML5. They clearly describe the role or meaning of the content they contain, making web pages easier to read and understand — both for developers and browsers.

  • <header>: Represents introductory content, like a page title or navigation links.
  • <footer>: Contains footer content, like contact info, copyright, or related links.
  • <main>: Represents the dominant content of the document, excluding headers, footers, or sidebars.
  • <section>: Defines a thematic grouping of content, typically with a heading.
  • <article>: Used for self-contained content like blog posts, news articles, or user comments.

Benefits of semantic elements:

  • Enhance SEO by giving content structure
  • Make your HTML more readable and maintainable

Example:

<article>
  <header>
    <h1>Understanding Semantic HTML</h1>
  </header>
  <p>Semantic HTML helps convey the meaning of content...</p>
  <footer>Written by Alex</footer>
</article>

🌐 Networking Basics

IP Address 💻

A unique number assigned to a device on a network.

MAC Address 🆔

A hardware identifier for network interfaces.

Router 📡

Forwards data packets between networks.

Switch 🔄

Connects multiple devices in a network.

Hub 🔌

A simple device that broadcasts data to all connected devices.

How Internet Works 🌐

The internet is a network of networks connected using IP, routing, and DNS.

DNS & Resolution 🖥️➡️📍

DNS (Domain Name System) translates domain names like example.com into IP addresses.
